# RemoveWindowsAI 使用指南

## 📌 根本原因分析

### 问题现象
在本地直接运行 `RemoveWindowsAi.ps1` 时，PowerShell 报出大量语法错误（如 `ParserError`, `UnexpectedToken`），但通过远程命令 `& ([scriptblock]::Create((irm ...)))` 却能正常运行。

### 根本原因
**Windows PowerShell 5.1 对“无 BOM 的 UTF-8 脚本”存在读取兼容性问题。**

1. **编码识别错误**：
   - 原始脚本文件是 **UTF-8 无 BOM** 格式。
   - 当使用 `-File` 参数在本地运行脚本时，Windows PowerShell 5.1 经常将其误判为 **ANSI** 编码读取。
   - 这导致脚本中的非 ASCII 字符（如特殊符号、注释等）被错误解码，破坏了后续的字符串和 Here-String 结构，从而引发连锁的语法解析错误。

2. **为何远程命令能跑通？**
   - 远程命令通过 `Invoke-RestMethod` 获取内容时，HTTP 响应头通常包含正确的编码信息，或者 PowerShell 在内存中处理字符串时使用了正确的解码逻辑。
   - 因此，内存中的脚本内容是完整的，能够正常执行。

### 解决方案
将脚本文件保存为 **UTF-8 with BOM** 格式。BOM（字节顺序标记）能明确告诉 PowerShell 这是一个 UTF-8 文件，从而强制其使用正确的编码读取。

## 🚀 如何运行脚本

### 前置要求
1. **必须使用 Windows PowerShell 5.1**（不支持 PowerShell 7+）。
2. **必须以管理员身份运行**。

### 运行命令
打开 **Windows PowerShell (管理员)**，执行以下命令：

#### 1. 交互式运行（带 UI 界面）
```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "D:\Code\RemoveWindowsAI\RemoveWindowsAi.ps1"
```

#### 2. 静默运行（全自动执行所有选项）
```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "D:\Code\RemoveWindowsAI\RemoveWindowsAi.ps1" -nonInteractive -AllOptions
```

#### 3. 指定选项运行
```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "D:\Code\RemoveWindowsAI\RemoveWindowsAi.ps1" -nonInteractive -Options DisableRegKeys,RemoveAppxPackages
```

## 🔄 如何更新脚本

当 GitHub 上的脚本有更新时，请使用提供的修复工具 `fix-encoding.ps1` 来下载并自动修复编码。

### 更新命令
在 **Windows PowerShell (管理员)** 中执行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "D:\Code\RemoveWindowsAI\fix-encoding.ps1" -OverwriteOriginal
```

**参数说明：**
- `-OverwriteOriginal`：自动覆盖本地的 `RemoveWindowsAi.ps1`，确保始终使用修复过编码的最新版本。
